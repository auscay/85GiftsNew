## Code of Conduct

Everyone should take ownership of the project and everycode written. This will help us move faster and build the best product we can.
Don't shy away from suggestions or making changes. If you think you have an Idea or better way to solve a problem you can build it and then show others on the team so we can decide if we accept it or not.

## How to Contribute

- **Issues**: If you find a bug, please you can make change to fix the bug, if it's outside your scope you can report it if you think it's taking you too long you can put up a notice so we put heads together to solve the bug..
- **Documentation**:Your code is also part of the documentation at the very least write readable code. use comments judiciously.
  
## Guidelines for Contributions

- Write clear and descriptive commit messages.

## Your First Code Contribution

1. Fork the repository.
2. Clone your fork locally.
3. Do 'npm install'.
4. Create a new branch for your feature or fix.
5. Make your changes and commit them.
6. Push your changes and submit a pull request.
